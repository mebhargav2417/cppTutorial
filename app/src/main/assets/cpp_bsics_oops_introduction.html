Introduction

     Problems with Structured Oriented Programming or Procedure Oriented Programming leads to Object Oriented Programming Development. As we know languages like C, COBOL, FORTRON commonly known as Procedure Oriented Programming languages. Generally procedure oriented programming consists of writing list of instructions into groups known as functions. Some of the characteristics of procedure oriented programming language is listed below.

  1. Large programs are divied into small programs called functions.
  2. Mostly functions share global data.
  3. Follows top down approach.
  4. Data moves freely from function to function around the system. So no restriction access available.
     Rather than procedure oriented programming, object oriented programming does not share data freely around the system. It considers data as a critical element. It combines data and functions as single unit known as Object. So these data and functions are binding up of into single unit, with the help of objects we will access those data and functions. The following are the concepts of Object oriented programming language.
  1. Classes
  2. Objects
  3. Inheritance
  4. Polymorphism
  5. Data abstraction
  6. Data encapsulation
  7. Message passing
